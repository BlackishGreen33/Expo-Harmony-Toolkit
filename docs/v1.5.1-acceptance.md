### 3. Debug HAP 构建

执行：

```bash
pnpm run harmony:build:debug
```

结果：

- 未通过。
- 问题拆成了两个阶段：

1. 初始失败点

- `ohpm install` 无法直接安装 npm 包内分发的本地 `*.har`。
- 原因是这些 `*.har` 实际上是带 `package/` 前缀的 tarball，`ohpm` 本地安装时期望根目录直接存在 `oh-package.json5`。

2. 修正后失败点

- toolkit 现在会在 `build-hap` 前先把本地 `*.har` 规范化展开到 `harmony/.expo-harmony/ohpm-local-deps/`，`ohpm install` 随后可通过。
- `hvigor assembleHap` 继续失败，进入 ArkTS 编译阶段后出现 adapter / RNOH 接口不匹配。
- 关键现象：
  - `@react-native-oh-tpl/react-native-gesture-handler` 当前包内 `oh-package.json5` 依赖的是 `@rnoh/react-native-openharmony: ^0.72.32`
  - 本仓库公开矩阵固定的是 `@react-native-oh/react-native-harmony@0.82.18`
  - 编译错误集中在 `RNGestureHandlerButton.ts`、`RNGestureHandlerRootView.ts`、`RNGestureHandlerModule.ts` 与 `RNOHPackagesFactory.ets`

当前判断：

- `Build Debug Hap(s)` 的最终阻塞已经不是本机环境问题，而是当前公开 matrix 下 UI-stack adapter 与 RNOH 运行时的兼容性问题。
