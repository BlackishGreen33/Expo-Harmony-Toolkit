# v1.9.0 验收记录（Bare Workflow Baseline + App Foundation Modules）

- 版本：`1.9.0`
- 发布轨：`next`
- 范围：repo 内可验证的 bare intake、app foundation preview shim、`react-native-gesture-handler` formal experimental slice

## 结论

`v1.9.0` 完成下一阶段主线 baseline，但不放宽 `verified`：

- bare workflow 进入 `coverageProfile=bare` 的 intake / debug baseline
- `expo-secure-store`、`expo-asset`、`expo-device`、`expo-clipboard`、`expo-haptics` 进入 `preview`
- 五个 app foundation modules 当前均为 `runtimeMode=shim`
- `react-native-gesture-handler` 进入 formal `experimental` adapter slice
- `latest` 仍只承接完整验收的 `verified` 能力

## Repo 验收项

- `fixtures/app-foundation-modules-app` 覆盖五个官方基础模块的 doctor / sync-template metadata
- `examples/official-native-capabilities-sample` 增加 `/secure-store`、`/asset`、`/device`、`/clipboard`、`/haptics` route
- `fixtures/gesture-handler-app` 增加 canonical package 与 Harmony adapter pairing
- `fixtures/bare-app` 保留 `android/`、`ios/` shape，并增加最小 JS entry 供 bundle/debug baseline 使用
- `DoctorReport.capabilities[]` 继续复用既有 `supportTier`、`runtimeMode`、`evidence`、`evidenceSource` 字段

## 边界

- 本记录不关闭 v1.8.x 四项 preview capability 的单 capability 真机 / release HAP evidence
- foundation modules 的 `runtimeMode=shim` 不代表真实 native parity
- `react-native-gesture-handler` 的 formal slice 只代表它不再是矩阵外模糊探索，不代表 verified
- bare workflow 只进入 intake / debug baseline，不代表 release-ready
