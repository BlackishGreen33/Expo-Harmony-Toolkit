# v1.8.3 验收记录（RNOH 0.82 patch 对齐）

- 版本：`1.8.3`
- 发布轨：`latest`
- 目标：将 RNOH runtime / CLI 从 `0.82.18` 对齐到当前 npm 最新 `0.82.29`

## 结论

`v1.8.3` 是兼容矩阵 patch 版本：

- `@react-native-oh/react-native-harmony`：`0.82.29`
- `@react-native-oh/react-native-harmony-cli`：`0.82.29`
- 公开矩阵仍是 `expo55-rnoh082-ui-stack`
- React Native 仍锁定 `0.82.1`

npm 上当前没有可用的 RNOH `0.83.x` 包，且 `@react-native-oh/react-native-harmony@0.82.29` 仍声明 `react-native@0.82.1` peer requirement。因此本版本不宣称 RN `0.83.x` 已进入 Harmony verified 矩阵。

## 验证命令

```bash
pnpm build
pnpm test
EXPO_HARMONY_RELEASE_SKIP_HAP=1 pnpm release:check
pnpm docs:sync
git diff --check
npm view expo-harmony-toolkit version dist-tags --json
```

## 边界

- 该版本只刷新 RNOH `0.82.x` patch 线，不新增 RN `0.83.x` runtime 承诺
- ccnubox 下游适配应依赖已发布的 `expo-harmony-toolkit@1.8.3`，不要使用本地 `link:../Expo-Harmony-Plugin`
- 任何 simulator install/start 证据都只能写作 app-shell 最低链路证据，不代表真机 verified
