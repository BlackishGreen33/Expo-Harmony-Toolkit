# v1.11.1 验收记录（首个公开 v1.11.x）

- 版本：`1.11.1`
- 发布轨：`latest`
- 日期：`2026-06-15`
- 范围：managed sidecar drift 诊断与 build-required drift 自动刷新口径

## 结论

`v1.11.1` 是第一个公开发布的 `v1.11.x`。`v1.11.0` 保留为未发布的 burn-down ledger checkpoint，不补 tag、不补 npm release、不补 GitHub Release。

本版本只收口 `sidecar-drift` 的 repo 内可测切片：

- 非 build-required managed sidecar drift 保留用户改动
- drift warning 固定输出 code：`sidecar.drift.requires-force`
- warning 明确提示 `expo-harmony sync-template --force` 或 `expo-harmony init --force`
- build-required autolinking 与 RNOH shim drift 可由 bundle/build gate 自动刷新
- 不新增 CLI 命令
- 不修改 `SyncResult` wire shape

## 公开边界

`v1.11.1` 发布到 `latest` 只代表稳定工具链 patch，不扩大能力承诺：

- 当前唯一公开 `verified` runtime matrix 仍是 `expo55-rnoh082-ui-stack`
- preview / experimental capability 不因为本版本发布而升级
- debug / release HAP build pass 不等于真机 device pass，也不等于 runtime pass
- catalog 外项目继续只承诺 intake 分类、blocker 说明和有序下一步

## Sidecar drift 验收

| 场景 | 预期 |
| --- | --- |
| drifted `metro.harmony.config.js` | `sync-template` 跳过该文件，保留用户内容，并输出 `sidecar.drift.requires-force` |
| drifted build-required autolinking file | `bundle` / `build-hap` gate 自动刷新，不输出 unresolved drift warning |
| drifted non-build-required managed file | `bundle` / `build-hap` 保留用户内容，并提示使用 `--force` |
