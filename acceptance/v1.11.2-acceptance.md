# v1.11.2 验收记录（非实机 burn-down 台账与降级策略）

- 版本：`1.11.2`
- 发布轨：未发布；本记录只准备 v1.11.2 repo baseline，不创建 tag、不发布 npm、不创建 GitHub Release
- 日期：`2026-06-16`
- 范围：不依赖真机的 blocker accounting、release/runtime 缺口口径、bare / third-party 降级策略

## 结论

`v1.11.2` 继续作为稳定工具链 / 文档 patch，不扩大 `verified`：

- `v1.11.1` 已经收口 repo 内可测的 `sidecar-drift` 工具链行为
- 本轮只把剩余 v1.11.x blocker 写成可追踪台账与诊断口径
- 不新增 CLI 命令
- 不修改 `DoctorReport`、`CapabilityDefinition` 或 `nextActions[]` wire shape
- 不把 debug/release HAP build pass 表述成真机 device pass 或 runtime pass
- 不把 preview / experimental capability 提升到 `verified`

## 非实机 burn-down 状态

| Track | v1.11.2 状态 | 后续出口 |
| --- | --- | --- |
| `release-evidence` | 四项 preview adapter capability 仍缺逐项 release HAP runtime acceptance | 每项 capability 单独关闭 release 记录；combined sample smoke 不可替代 |
| `bare-release-gap` | bare workflow 继续只有 intake / debug baseline | release HAP 与 runtime evidence 闭环前，不得宣称 release-ready |
| `third-party-native` | Wave A / Wave B 继续停留在 `experimental` | 关闭 device / release evidence；无法关闭时必须写明 fallback / replacement path |
| `sidecar-drift` | repo 内可测切片已由 v1.11.1 收口 | 真实项目 drift 若无法关闭，写入 accepted exception，不能替代 runtime evidence |

## 降级与替代路径

- preview adapter capability：继续保留 `preview`，对外只承诺 bundle/debug build baseline 与已有人工记录；release HAP runtime 未关闭前不进入 `verified`
- bare workflow：继续保留 `coverageProfile=bare` intake path，保留 native directories，使用 `doctor` 分类；release HAP/runtime 未关闭前不称为 release-ready
- Third-party Wave A：继续要求 `doctor --target-tier experimental`，async-storage / screens 走 Harmony adapter，safe-area 走 toolkit shim；证据关不掉时记录替代依赖或功能降级路径
- Third-party Wave B：继续要求 `doctor --target-tier experimental`，WebView / media-library / Lottie / Skia 走 adapter，JPush runtime evidence 单独记录；证据关不掉时按 surface 写出 fallback
- catalog 外 native dependency：继续按 `third-party-native-gap` 报出，不能降写成普通 matrix drift

## 验收边界

- 本记录不要求真机 device evidence
- 本记录不关闭 release HAP runtime evidence
- 本记录不发布 `1.11.2`
- 当前 npm `latest` 与 GitHub Release 仍以已发布版本为准，直到后续显式发版
