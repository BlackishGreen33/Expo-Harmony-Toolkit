- 版本：`1.11.0`
- 发布轨：未定；本记录是 v1.11.x 起手台账，不是发布记录

## 结论

`v1.11.0` 第一阶段只建立 burn-down ledger，不扩大 `verified`：

- `latest` 仍只承诺完整验收的 `verified` 能力
- 当前唯一公开 `verified` runtime matrix 仍是 `expo55-rnoh082-ui-stack`
- preview / experimental capability 不因为进入 ledger 就升级
- debug / release HAP build pass 不等于真机 device pass，也不等于 runtime pass
- catalog 外项目继续只承诺 intake 分类、blocker 说明和有序下一步

## Burn-down ledger

| Track | 当前状态 | 缺口 | v1.11.x 出口 |
| --- | --- | --- | --- |
| `release-evidence` | preview adapter capability 已有 bundle / debugBuild evidence，部分已有 manual device baseline | per-capability release HAP runtime evidence 仍未逐项关闭 | 每个 capability 单独记录 release acceptance；combined sample smoke 不可替代单项证据 |
| `bare-release-gap` | bare workflow 已可被 `doctor` 分类，并保持 intake / debug baseline | release HAP 与 runtime evidence 未闭环 | 继续标记为非 release-ready，直到 release HAP 与 runtime 证据补齐 |
| `third-party-native` | Wave A / Wave B 已进入 `experimental` formal onboarding | device / release evidence 仍缺，部分 adapter runtime 行为仍待验证 | 逐项补证据；无法关闭的项写明降级策略或替代路径 |
| `sidecar-drift` | catalog-covered managed 项目已有 `sync-template -> bundle -> build-hap --mode debug` gate | 真实项目 sidecar drift 仍可能发生 | drift 必须被明确报出、可重复修复，或记录为 v2.0 前已知例外 |

## Sidecar drift audit

v1.11.0 第一项可验收切片先关闭 `sidecar-drift` 的 repo 内可测部分：

| Audit result | 含义 | 当前处理 |
| --- | --- | --- |
| `detected` | managed sidecar 文件与上次生成 manifest 不一致 | `sync-template` / `init` / `bundle` / `build-hap` 输出稳定 warning code：`sidecar.drift.requires-force` |
| `auto-refreshed build-required files` | bundle/build gate 必需的 autolinking 与 RNOH shim 文件发生 drift | build gate 允许自动刷新这些文件，不把它们误报成 unresolved drift |
| `requires --force` | 非 build-required managed sidecar 文件发生 drift | 跳过写入，保留用户改动，并提示 `expo-harmony sync-template --force` 或 `expo-harmony init --force` |
| `accepted exception` | 无法在 v1.11.x 关闭的真实项目 drift | 必须写入后续 acceptance 记录，不能伪装成 build/device/runtime pass |

本切片不新增 CLI 命令，不改变 `SyncResult` wire shape，只稳定 warning 文案与测试口径。
