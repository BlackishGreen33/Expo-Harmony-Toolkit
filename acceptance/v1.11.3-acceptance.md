# v1.11.3 验收记录（v2 readiness 非实机收口）

- 版本：`1.11.3`
- 发布轨：repo-only update；本记录不创建 tag、不发布 npm、不创建 GitHub Release
- 日期：`2026-06-18`
- 范围：把 v2 前剩余 blocker 收敛成明确 gate、fallback、exception 与公开边界，不依赖真机，不关闭 per-capability release runtime evidence

## 结论

`v1.11.3` 是 v2 readiness 的非实机收口 patch，不扩大 `verified`：

- 不新增 CLI 命令
- 不修改 `DoctorReport`、`CapabilityDefinition` 或 `nextActions[]` wire shape
- 不把 preview / experimental capability 提升到 `verified`
- 不把 debug/release HAP build pass、simulator install/start 或 app-shell smoke 表述成真机 device pass 或 per-capability runtime pass
- `v1.11.2` 仍是当前已发布的 tag/npm/GitHub Release surface；本记录只更新仓库代码与文档台账

## 降级与替代路径

- preview adapter capability：保持 `preview`，对外只承诺 bundle/debug build baseline 与已有人工记录；release HAP runtime 未逐项关闭前不进入 `verified`
- app foundation modules：保持 `preview` + `runtimeMode=shim`，不宣称 native parity
- bare workflow：保持 `coverageProfile=bare` intake path，保留 native directories，并继续使用 `doctor` 输出 release/runtime blocker
- Third-party Wave A：async-storage / screens 走 Harmony adapter，safe-area 走 toolkit shim；证据关不掉时记录替代依赖或功能降级
- Third-party Wave B：WebView / media-library / Lottie / Skia 走 adapter，JPush 与 notifications delivery evidence 单独记录；证据关不掉时按 surface 写出 fallback
- catalog 外 native dependency：继续按 `third-party-native-gap` 报出，不降写成普通 matrix drift

## 验收边界

- 本记录不要求真机 device evidence
- 本记录不关闭 release HAP runtime evidence
- 本记录不发布 `1.11.3`
- 本记录不改变 npm `latest` / `next` dist-tag 状态
- 本记录不改变 true-device / per-capability runtime evidence 要求
