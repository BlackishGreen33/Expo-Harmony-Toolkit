# v1.9.2 验收记录（Third-party Native Wave A）

- 版本：`1.9.2`
- 发布轨：`latest`
- 范围：repo 内可验证的 Third-party Native Wave A formal experimental onboarding，以及 `v1.9.1` 后的 build-hap 安全基线

## 结论

`v1.9.2` 把第一批高频第三方 native blocker 拉进正式 telemetry，但不放宽 `verified`：

- `@react-native-async-storage/async-storage` 进入 formal `experimental` adapter slice
- `react-native-screens` 进入 formal `experimental` adapter slice
- `react-native-safe-area-context` 进入 formal `experimental` shim slice
- `doctor --strict` 与 `doctor --target-tier preview` 仍阻断 Wave A
- `doctor --target-tier experimental` 在必要 adapter 配对满足时允许 Wave A
- `latest` 仍只承诺完整验收的 `verified` 能力

| 项目 | 层级 | runtimeMode | bundle | debugBuild | device | release |

| `react-native-safe-area-context` | `experimental` | `shim` | `yes[automated]` | `yes[automated]` | `no[none]` | `no[none]` |

## Repo 验收项

- `fixtures/third-party-wave-a-app` 覆盖 Wave A canonical packages 与 adapter 配对
- `fixtures/third-party-wave-a-missing-async-storage-adapter-app` 覆盖 async-storage adapter 缺失诊断
- `fixtures/third-party-wave-a-missing-screens-adapter-app` 覆盖 screens adapter 缺失诊断
- `DoctorReport.capabilities[]` 继续复用既有 `supportTier`、`runtimeMode`、`evidence`、`evidenceSource` 字段
- `react-native-safe-area-context` 继续通过 `.expo-harmony/shims/react-native-safe-area-context` 走 toolkit-managed shim

## 边界

- 本记录不关闭 Wave A 的 device / release HAP runtime evidence
- async-storage 的真实持久化、迁移、release 行为仍需单独验收
- screens 的 navigation-stack runtime 与 adapter autolinking 稳定性仍需单独验收
- safe-area 的真实 native inset measurement 仍未进入 verified
- `latest` 发布本版本不代表 Wave A 进入 `verified`
