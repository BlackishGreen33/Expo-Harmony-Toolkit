对象：

执行：

```bash
node ../../bin/expo-harmony.js doctor --project-root . --strict
node ../../bin/expo-harmony.js init --project-root .
node ../../bin/expo-harmony.js bundle --project-root .
node ../../bin/expo-harmony.js build-hap --project-root . --mode debug
```

结果：

- `doctor --strict`：通过，矩阵识别为 `expo55-rnoh082-ui-stack`。
- `init`：通过。
- `bundle`：通过。
- `build-hap --mode debug`：通过，产出：

```text
examples/official-ui-stack-sample/harmony/entry/build/default/outputs/default/entry-default-unsigned.hap
```

设备侧验证：


当前判断：

### 3. Preview Native Capabilities 验证

对象：

- `examples/official-native-capabilities-sample`

执行：

```bash
node ../../bin/expo-harmony.js doctor --project-root . --target-tier preview
node ../../bin/expo-harmony.js init --project-root .
node ../../bin/expo-harmony.js bundle --project-root .
node ../../bin/expo-harmony.js build-hap --project-root . --mode debug
```

结果：

- `doctor --target-tier preview`：通过，`expo-file-system` / `expo-image-picker` 识别为 `manual/preview`。
- `init`：通过，并生成 Harmony permission declaration 与 preview shim。
- `bundle`：通过。
- `build-hap --mode debug`：通过，产出：

```text
examples/official-native-capabilities-sample/harmony/entry/build/default/outputs/default/entry-default-unsigned.hap
```

设备侧验证：

```bash
hdc install -r entry-default-unsigned.hap
hdc shell aa start -a EntryAbility -b com.blackishgreen.expoharmonynativepreview
```

结果：

- HAP 安装成功。
- `EntryAbility` 启动成功。
- `bm dump` 可见 request permissions：
  - `ohos.permission.CAMERA`
  - `ohos.permission.READ_IMAGEVIDEO`
  - 对应 `$string:` reason 已写入 bundle metadata。
- 通过 `uitest dumpLayout` 与点击事件验证：
  - 首页可见 `Official Native Capabilities Sample`
  - `/file-system` 页面可见 `expo-file-system preview`
  - 点击 `Attempt preview write` 后，message 更新为 preview bridge 文案，不 crash
  - `/image-picker` 页面可见 `expo-image-picker preview`
  - 点击 `Attempt preview pick` 后，message 更新为 preview bridge 文案，不 crash

当前判断：

- preview tier 的承诺已经达到：
  - 可被 `doctor` 分级识别
  - 可 `init`
  - 可 `bundle`
  - 可 `build-hap --mode debug`
  - 可安装与启动
  - 点击 capability demo 不会 crash，而是稳定落到 preview bridge 提示

## 非阻塞观察

- `env.signing.missing` 仍然存在；当前 sidecar 没有 release signing config，这不阻塞 npm 发版，也不阻塞 debug HAP 验证。
- 设备日志里仍可看到一些非致命 debug 噪音，例如：
  - `StatusBar is translucent by default`
  - `DevMenu` / `ACCELEROMETER` 相关提示

- 本轮没有观察到阻止应用启动、阻止页面渲染、或导致 preview / verified sample route 崩溃的 fatal runtime 问题。
