# v1.8.0 验收记录（主线重排与 intake hardening 版）

- 版本：`1.8.0`
- 日期：`2026-04-04`
- 目标：把 roadmap 从“主流 Expo 项目覆盖”进一步收紧到 `v2.0.0 = 任何 Expo 项目都能可靠打包成鸿蒙 App`，同时把 v1.8.0 需要的 intake hardening、capability board、第三方 native blocker 追踪链先做成 repo 内部事实

## 结论

`v1.8.0` 的正确结论不是“任何 Expo 项目已经全部兼容”，而是：

- `doctor` 现在已经能把项目分进 `managed-core`、`managed-native-heavy`、`bare`、`third-party-native-heavy`
- `doctor-report.json` 与 `toolkit-config.json` 现在会同步输出：
  - `coverageProfile`
  - `dependencies[].gapCategory`
  - `nextActions`
- bare workflow、`react-native-gesture-handler`、未知第三方 native module 不再停留在模糊灰区，而是进入可追踪的主线 blocker 叙事
- 官方 native sample 从“单一 combined smoke”升级成“combined smoke + per-capability acceptance board”
- roadmap 已经把 `v2.0.0` 直接定义成“任何 Expo 项目都能可靠打包成鸿蒙 App”，并把中间过程拆成更多小版本

## 这轮主线变更

- doctor / metadata
  - 新增 `coverageProfile`
  - 新增 `gapCategory`
  - 新增有序 `nextActions`
  - `toolkit-config.json` 与 CLI report 同步输出这套诊断信息
- fixture / regression
  - 新增 `verified-app`
  - 新增 `bare-app`
  - 新增 `gesture-handler-app`
  - 新增 `third-party-native-gap-app`
- preview capability tracking
  - 新增 `v1.8.x-capability-board`
  - 为 `expo-file-system`、`expo-image-picker`、`expo-location`、`expo-camera` 各自建立 device / release acceptance 模板
- roadmap / docs
  - 把 `v2.0.0` 目标升级成“任何 Expo 项目可靠打包”
  - 拆出 `v1.9.1`、`v1.9.2`、`v1.10.0`、`v1.11.0`
