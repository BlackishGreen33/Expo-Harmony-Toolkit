# v1.10.0 验收记录（Any-project Intake Freeze）

- 版本：`1.10.0`
- 发布轨：`latest`
- 范围：`doctor` intake 分类、blocker 类型、`nextActions[]` 顺序、v1.10 regression fixtures、文档口径

## 结论

`v1.10.0` 冻结 Any-project Intake，不扩大 `verified`：

- 任意 Expo 项目进入 `doctor` 后必须能落到 `coverageProfile`
- unsupported / unknown / native-risk / bare / official-module 情况必须有明确 `gapCategory`
- `nextActions[]` 顺序固定为：版本矩阵问题 -> 项目形态问题 -> native blocker -> unknown JS/native 依赖 -> build gate
- catalog-covered managed fixture 可以进入 `sync-template -> bundle -> build-hap --mode debug` 验收路径
- catalog 外 fixture 只要求 doctor 分类和 nextActions 完整，不要求 build 成功
- `latest` 仍只承诺完整验收的 `verified` 能力

## Regression fixtures

| Fixture | 预期 |
| --- | --- |
| `fixtures/v110-catalog-managed-app` | `coverageProfile=managed-core`，无 blocking issues，进入 build gate |
| `fixtures/v110-bare-app` | `coverageProfile=bare`，不误报成 `managed-core` |
| `fixtures/v110-unknown-js-only-app` | unknown JS-only 依赖为 `buildabilityRisk=js-only-unknown`，保留低风险 bundle 验证建议 |
| `fixtures/v110-unknown-native-looking-app` | unknown native-looking 依赖为 `buildabilityRisk=native-risk` 与 `gapCategory=third-party-native-gap` |
| `fixtures/v110-mixed-intake-app` | official module、third-party native、unknown JS/native dependency 同时报出，`nextActions[]` 顺序稳定 |

## Repo 验收项

- `DoctorReport.coverageProfile` 继续只使用 `managed-core`、`managed-native-heavy`、`bare`、`third-party-native-heavy`
- `dependencies[].gapCategory` 继续只使用 `matrix-drift`、`official-module-gap`、`third-party-native-gap`、`bare-workflow-gap`
- bare workflow 不进入 verified 承诺
- 官方 preview/experimental module 不进入 verified 承诺
- unknown native-looking dependency 不再被模糊归为普通 matrix drift
- unknown JS-only dependency 不阻断，但仍保留 catalog 外提示

## Catalog-covered build gate

```bash
pnpm install --ignore-scripts
```

- `sync-template`：`written: 55`，`skipped: 0`
- `bundle`：`Status: succeeded`
- `build-hap --mode debug`：`Status: succeeded`
- HAP artifact：`harmony/entry/build/default/outputs/default/entry-default-unsigned.hap`

## 边界

- 本记录不提升任何 capability 到 `verified`
- 本记录不要求真机 device evidence
- 本记录不要求 catalog 外项目全部完成可靠打包
- 本记录不关闭 v1.8.x 单 capability device / release evidence
- 发布动作另行执行；本记录只说明 v1.10.0 implementation / acceptance 准备
